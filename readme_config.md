# Структура конфигов config.yaml и secret_config.yaml

## config.yaml - конфиг с открытыми данными

### 1. Тестовое окружение:
* Среда SDP для выполнения тестов (_production_ или _test_)

#### 1.1. URL:
* Адрес, на котором проходят тесты. Включает в себя доменное имя + порт + /

#### 1.2. МРФ:

##### 1.3. Список всех МРФ:
* МРФ, на котором выполняются тесты

* Прописаны отдельно для тестовой и продовой сред

* Внутри каждого МРФ:

    - _sdp_api_url_ - ссылка, по которой доступны методы SDP.
        Включает в себя доменное имя + порт + /
        
    - Отели. _main_hotel_ - в SDP сущность клиент, основной отель для
        проведения большинства тестов. _second_hotel_ - второй отель,
        используется в некоторых тестах
        
        * _hotel_id_ - ID клиента в SDP
        
        * Комнаты: _main_room_ - основная комната, на которой проводятся
            большинство тестов, а также еще две комнаты, которые
            используются в некоторых тестах. _third_room_ - в данной
            комнате должны быть следующие покупки:
            
            1. "Сахара"
            1. "Карлик Нос"
            1. "Взрослый Деловой"
            
            - _san_ - номер учетной записи в SDP (SAN).
                Учетная запись == комната.
                
            - _number_ - номер комнаты. Задается в SDP в дополнительных
                настройках.
                
            - _sdp_room_id_ - ID учетной записи в SDP.
                **Не путать с SAN**
                
            - _connection_slots_ - подключения, которые доступны
                в учетной записи в SDP.
                
                * Всего должно быть 4 подключения:
                
                    - _first_slot_ - приставка STB
                    - _second_slot_ - Smart TV
                    - _third_slot_ - Неизвестное устройство (MAC-адрес
                        не соответствует ни STB, ни Smart TV)
                    - _fourth_slot_ - свободный слот (MAC-адрес пустой)


## secret_config.yaml - конфиг с зашифрованными данными

### 1. Тестовое окружение:
* Среда SDP для выполнения тестов (_production_ или _test_)

#### 1.1. URL:
* Адрес, на котором проходят тесты. Включает в себя доменное имя + порт + /

#### 1.2. МРФ:

##### 1.3. Список всех МРФ:
* МРФ, на котором выполняются тесты

* Прописаны отдельно для тестовой и продовой сред

* Внутри каждого МРФ:

    - _sdp_auth_data_ - авторизационные данные SDP. Необходимы для проверок
        некоторых тестов непосредственно через методы SDP. Включают в себя
        логин, пароль и язык (можно всегда ставить "ru").
        
    - _role_auth_data_ - авторизационные данные пользователей
        Hotel TV Manager. Данные пользователи должны быть созданы заранее
        и их данные должны быть занесены в secret_config. Данные для каждого
        пользователя: email, пароль и ID.
        
        * _admin_mrf_ - администратор Ростелеком. Он один на весь МРФ.
        
        * _main_hotel_users_ - пользователи главной гостиницы
            (сотрудник и администратор гостиницы).
            
        * _second_hotel_users_ - пользователи второй гостиницы
            (сотрудник и администратор гостиницы).
            
## **Важная информация:**

* На данный момент полностью заполненным и готовым для проведения тестов
    является МРФ Центр тест (ct, test). Для проведения тестов в другом
    тестовом окружении необходимо настроить оба конфига по аналогии
    с **ct test**.
